3. Tag System & Groups
Tags are assigned to units in F3 mode using CTRL+T. Tags are the primary way to select units in triggers. A unit can have multiple tags. Tags float above units as green text when assigned.
Assigning Tags
// In F3 mode:
1. Select unit(s)
2. Press CTRL+T
3. Type tag name → press Add
4. Green tag text appears above unit

// A unit can hold multiple tags simultaneously
// Tags survive unit death for detection purposes
// BUT: blown-up units may lose tags — use zone+state checks as backup
Standard Tag Naming Convention — This Campaign
// IMPERIAL SIDE
IMP_CPY_CMD ← Company Commander / Captain (1 man)
IMP_PLT1_LT ← 1st Platoon Lieutenant (1 man)
IMP_PLT1_SQD_A ← 1st Platoon, Squad Alpha (9 men)
IMP_PLT1_SQD_B ← 1st Platoon, Squad Bravo (9 men)
IMP_PLT2_LT ← 2nd Platoon Lieutenant (1 man)
IMP_PLT2_SQD_A ← 2nd Platoon, Squad Alpha (9 men)
IMP_PLT2_SQD_B ← 2nd Platoon, Squad Bravo (9 men)
IMP_CPY_FALLBACK ← Waypoint: company fallback point
IMP_PLT1_FALLBACK ← Waypoint: platoon 1 fallback point

// REBEL SIDE
REB_CMD ← Rebel cell leader / SpecForce captain (1 man)
REB_TEAM_A ← Strike team Alpha
REB_TEAM_B ← Strike team Bravo
REB_EXTRACT ← Waypoint: extraction point
REB_FALLBACK ← Waypoint: fallback point

// SAW GERRERA SPECIAL
SAW_PARTISANS ← Saw's men as a group
SAW_EXTRACT ← Waypoint: Saw's extraction point (separate from Rebel extract)
Waypoints are not units
FALLBACK and EXTRACT tags refer to waypoints placed in F3 → Waypoints, not tagged units. Place the waypoint on the map first, then reference it by name in actor_to_waypoint commands.
4. Entities Condition
The entities condition checks the count or state of tagged units. Used to detect deaths, alive counts, and trigger thresholds. This is the primary condition for morale break triggers.
Basic Death Detection
// Fires when the tagged unit is confirmed dead
entities
  source: standard
  tag: IMP_PLT1_LT
  state: dead
  count: >= 1
Advanced — Check Alive Count (for Saw Rogue trigger)
// Fires when fewer than 4 of Saw's men are still alive
entities
  source: advanced ← right-click source → select advanced
  selector: group
    select: SAW_PARTISANS
    include: prop: human
    exclude: state: dead
    exclude: state: inactive
  count: <= 4 ← adjust to ~30% of Saw's starting headcount
Blown-up units bug
Units killed by explosions may disappear completely, removing their tag. This means a dead count of >= 1 may not fire if the unit was blown up. Use the alive count approach (exclude dead, check count falls below threshold) as a more reliable alternative for critical triggers.
5. Actor State Parameters
actor_state changes AI behavior for tagged units mid-mission. You can overwrite a previous actor_state at any time. Only include parameters you want to change — unspecified parameters retain their current values.
ParameterValuesEffectStatus
control AI / player Who controls this unit — always set to AI for scripted behavior CONFIRMED
move_mode free / hold / assault free = seeks cover, moves tactically. hold = stays in position. assault = advances aggressively CONFIRMED
no_retreat on / off on = unit will NOT fall back regardless of odds. off = AI makes its own retreat decisions based on ratios CONFIRMED
advance_ratio number (e.g. 0.1 to 5) AI advances if its allies outnumber enemies by this ratio. 0.1 = attack even heavily outnumbered. 5 = only advance if 5:1 advantage CONFIRMED
retreat_ratio number (e.g. 0.5 to 5) AI retreats if enemies outnumber allies by this ratio. 0.5 = retreat even when barely outnumbered. 5 = hold until outnumbered 5:1 CONFIRMED
fire_mod Open / hold_fire / return Open = shoot freely. hold_fire = do not shoot. return = only return fire CONFIRMED
speed Bow / Assault / Walk / Crouch Movement speed. Bow is fastest (sprint). Assault is combat run. Crouch is stealth movement CONFIRMED
no_advance on / off When on, unit will not advance toward enemies. Stays at current position and fires CONFIRMED
Example — Set Units to Hold and Not Retreat
actor_state
  selector: tag IMP_PLT1_SQD_A
  control: AI
  move_mode: hold
  no_retreat: on
  fire_mod: Open
  advance_ratio: 5 ← only advance if heavily outnumbering enemy
Example — Set Units to Fall Back Under Pressure
actor_state
  selector: tag IMP_PLT1_SQD_A
  control: AI
  move_mode: free
  no_retreat: off
  retreat_ratio: 0.5 ← retreat even when barely outnumbered
6. Actor to Waypoint
Sends tagged units to move toward a specific waypoint. Used to direct retreating units to fallback positions.
ParameterValuesEffectStatus
selector tag [TAG_NAME] Which units to send CONFIRMED
waypoint [WAYPOINT_NAME] Destination waypoint CONFIRMED
drop_orders true / false When true, units abandon current orders and immediately move to waypoint CONFIRMED
sensor senseless Units ignore enemies and move directly to waypoint without stopping to fight CONFIRMED
approach teleport / walk How units reach waypoint. Walk = normal movement. Teleport = instant CONFIRMED
Example — Route Units to Fallback (senseless movement)
actor_to_waypoint
  selector: tag IMP_PLT1_SQD_A
  waypoint: IMP_PLT1_FALLBACK
  drop_orders: true
  sensor: senseless ← move without stopping to engage
sensor: senseless for retreat cinematics
Using senseless on retreating units means they run past enemies without engaging — this looks cinematic and realistic. Soldiers in full retreat don't stop to fight back.

8. Morale System
Morale is not a tracked number. It is a state determined by three campaign-layer conditions that are checked BEFORE the battle is set up in the editor. The morale state determines which trigger set you build into the mission.
The Three Morale Conditions
ConditionCheckSource
Supply Status Nominal / Reduced / Critical — already tracked in Sector Command document Sector Command document → planet supply field
Casualty Level Current headcount vs original headcount Sector Command document → force headcount field
Commander Presence Is a named commander assigned to this force this turn? Campaign layer — commander location tracking
Morale States
HIGH
Supply: Nominal
Strength: above 75%
Commander: present
Full force deployed. Use STEADY triggers. AI is aggressive. No retreat on LT death.
STEADY
Supply: Nominal
Strength: above 75%
Commander: absent
Full force deployed. Use STEADY triggers. Squads hold on LT death but stop advancing.
SHAKEN
Supply: Reduced
OR Strength: 50–75%
Reduced force deployed. Use BROKEN triggers. Squads rout on LT death.
BATTERED
Supply: Critical
OR Strength: below 50%
Heavily reduced force. Defensive only. Rout on any commander death.
BROKEN
Strength: below 25%
OR Critical supply + no commander
Cannot attack. Auto-retreats when engaged. Use extreme BROKEN triggers only.
Commander Effect on Morale
// A named commander present shifts morale threshold UP one level:
Shaken force + named commander = treated as Steady for trigger purposes
Battered force + named commander = treated as Shaken for trigger purposes

// No commander shifts threshold DOWN one level:
Steady force + no commander = treated as Shaken
Shaken force + no commander = treated as Battered
9. Morale Break Triggers
Build ONLY the trigger set that matches the force's morale state before the battle. Do not include both sets in the same mission — pick one based on the campaign state.

Each trigger fires only once (Trigger off). Delays are intentional — staggered timing makes retreats look like panic spreading rather than scripted simultaneous movement.
TRIGGER 1 — Lieutenant Killed (High / Steady Morale)
When to use: Supply Nominal + Strength above 75%. Squads are disciplined — they hold when the LT dies but stop advancing.
TRIGGER NAME: PLT1_LT_KIA_STEADY

// CONDITION
entities
  source: standard
  tag: IMP_PLT1_LT
  state: dead
  count: >= 1

// COMMANDS
actor_state
  selector: tag IMP_PLT1_SQD_A
  control: AI
  move_mode: hold ← stop advancing, hold position
  no_retreat: on ← do not fall back
  advance_ratio: 5 ← only advance if heavily outnumbering enemy

actor_state
  selector: tag IMP_PLT1_SQD_B
  control: AI
  move_mode: hold
  no_retreat: on
  advance_ratio: 5

Trigger off: PLT1_LT_KIA_STEADY

// RESULT: Squads stop advancing. Take cover. Fire from position.
// Looks like soldiers who lost their officer and are waiting for orders.
// Repeat this trigger structure for PLT2_LT, PLT3_LT etc.
TRIGGER 2 — Lieutenant Killed (Shaken / Battered Morale)
When to use: Supply Reduced OR Strength below 75%. Force is already under pressure — losing the LT breaks them.
TRIGGER NAME: PLT1_LT_KIA_BROKEN

// CONDITION
entities
  source: standard
  tag: IMP_PLT1_LT
  state: dead
  count: >= 1

// COMMANDS
delay: 2 ← brief pause before panic sets in

// Squad A — immediately routes to fallback
actor_state
  selector: tag IMP_PLT1_SQD_A
  control: AI
  move_mode: free
  no_retreat: off
  retreat_ratio: 0.5 ← retreat even if barely outnumbered

actor_to_waypoint
  selector: tag IMP_PLT1_SQD_A
  waypoint: IMP_PLT1_FALLBACK
  drop_orders: true
  sensor: senseless ← run past enemies without stopping

delay: 4 ← Squad B holds 4 more seconds before also breaking

// Squad B — routes after delay
actor_state
  selector: tag IMP_PLT1_SQD_B
  control: AI
  move_mode: free
  no_retreat: off
  retreat_ratio: 0.5

actor_to_waypoint
  selector: tag IMP_PLT1_SQD_B
  waypoint: IMP_PLT1_FALLBACK
  drop_orders: true

Trigger off: PLT1_LT_KIA_BROKEN

// RESULT: Squad A runs immediately. Squad B holds 4 seconds then breaks.
// Staggered timing makes panic look organic — not scripted simultaneous rout.
TRIGGER 3 — Company Commander Killed (Partial Collapse)
When to use: Any morale state. Losing the Captain hurts everyone but doesn't fully collapse the force. Some platoons hold, some break.
TRIGGER NAME: CPY_CMD_KIA

// CONDITION
entities
  tag: IMP_CPY_CMD
  state: dead
  count: >= 1

// COMMANDS
delay: 3

// PLATOON 1 — breaks immediately (closest to action / worst position)
actor_state
  selector: tag IMP_PLT1_SQD_A
  control: AI
  move_mode: free
  no_retreat: off
  retreat_ratio: 0.5

actor_to_waypoint
  selector: tag IMP_PLT1_SQD_A
  waypoint: IMP_CPY_FALLBACK
  drop_orders: true
  sensor: senseless

// PLATOON 2 — holds position (veterans / better cover)
actor_state
  selector: tag IMP_PLT2_SQD_A
  control: AI
  move_mode: hold
  no_retreat: on
  advance_ratio: 5

actor_state
  selector: tag IMP_PLT2_SQD_B
  control: AI
  move_mode: hold
  no_retreat: on
  advance_ratio: 5

// PLATOON 3 — holds 8 more seconds then also breaks
delay: 8

actor_state
  selector: tag IMP_PLT3_SQD_A
  control: AI
  move_mode: free
  no_retreat: off
  retreat_ratio: 0.5

actor_to_waypoint
  selector: tag IMP_PLT3_SQD_A
  waypoint: IMP_CPY_FALLBACK
  drop_orders: true

Trigger off: CPY_CMD_KIA

// RESULT: One platoon routes immediately. One holds throughout.
// One holds 8 seconds then breaks. Realistic command breakdown.
// NOT a full collapse — partial failure exactly like real leadership loss.
10. Saw Gerrera Rogue Trigger
Fires when Saw's Partisans are in a joint operation AND have taken heavy casualties. The result is determined by a dice roll BEFORE the battle is set up — the result determines how Saw's player assignment is configured in F2.
Pre-Battle Dice Roll (Campaign Layer)
// Roll 1d6 BEFORE building the Men of War scenario
// This determines how you assign Saw's player number in F2:

Roll 1-2: FRIENDLY FIRE
  // Assign SAW_PARTISANS to Player 2 (separate from Rebel Player 1)
  // AI treats all other factions as enemies — Saw's men fight everyone
  // Both allied Rebels AND Imperials take casualties from Saw's men

Roll 3-4: WITHDRAWAL
  // Keep Saw as Player 1 (same team as Rebels)
  // Trigger below fires — Saw's men extract and leave allies alone

Roll 5-6: HOLDS
  // No special trigger — Saw fights normally with allied Rebels
TRIGGER — Saw's Withdrawal (Roll 3-4 result only)
TRIGGER NAME: SAW_WITHDRAWAL

// CONDITION — fires when Saw's force has taken ~70% casualties
entities
  source: advanced
  selector: group
    select: SAW_PARTISANS
    include: prop: human
    exclude: state: dead
    exclude: state: inactive
  count: <= [30% of starting Saw headcount]

// COMMANDS — Saw's men extract, leaving allies to fight alone
actor_state
  selector: tag SAW_PARTISANS
  control: AI
  move_mode: free
  fire_mod: hold_fire ← stop shooting, just run

actor_to_waypoint
  selector: tag SAW_PARTISANS
  waypoint: SAW_EXTRACT ← Saw's own separate extraction point
  drop_orders: true
  sensor: senseless ← don't stop — just extract

Trigger off: SAW_WITHDRAWAL

// RESULT: Saw's men stop firing and extract to their waypoint.
// The Rebel allied cell is now fighting without Partisan support.
// Cinematically: Saw's men simply leave. Allies have to deal with it.
Friendly Fire Setup (Roll 1-2)
For the friendly fire result, no trigger script is needed. Simply assign SAW_PARTISANS units to Player 2 in F2 mode before loading the mission. The engine's faction system handles the rest — Player 2 is enemy to both Player 0 (Imperials) and Player 1 (Rebels). This is the cleanest implementation and requires no scripting.

12. Known Issues & Warnings
IssueCauseWorkaround
Unit vanishes when killed by explosion Explosion kills remove the entity entirely including its tag Use alive count check (exclude dead, count falls below threshold) instead of dead count check for explosion-prone commanders
Units stop after firefights and won't continue AI enters idle state after engaging Add ai_move: disable in actor_state, or re-issue actor_to_waypoint after the engagement zone
Nature tab empty in editor — no sky domes Running editor from AotR mod folder which lacks base game sky assets Copy Skydomes folder from base game Data\Art\ into AotR mod Data\Art\ directory
ALT+G crashes editor instantly Known editor bug — unknown cause Never press ALT+G. There is no fix. The editor will close without saving.
Triggers fire repeatedly Trigger not turned off after first execution Always add Trigger off: [TRIGGER_NAME] as the last command in any trigger that should only fire once
Units assigned wrong player fight wrong enemies Player numbers not set correctly in F2 before mission load Player 0 = Imperial. Player 1 = Rebel. Player 2 = Saw's Partisans (rogue roll only). Confirm in F2 before play.
Retreat trigger fires immediately at battle start Pre-existing casualties from campaign already below threshold Add an Event condition gate — set a start event after a delay so the threshold can't trigger before the battle is established
